# Введение в React: от чистого HTML к React-элементам

Давайте сперва создадим простую HTML разметку.

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta name="description" content="Описание вашего сайта" />
    <meta name="author" content="Ваше имя" />

    <title>Название сайта</title>

    <!-- Подключение favicon -->
    <link rel="icon" href="favicon.ico" type="image/x-icon" />

    <!-- Подключение CSS -->
    <link rel="stylesheet" href="styles.css" />

    <!-- Подключение шрифтов или библиотек -->
    <!-- <link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet"> -->
  </head>

  <body>
    <div id="root">
      <h1>Hello World!</h1>
    </div>

    <!-- Подключение JavaScript -->
    <script src="script.js">

      // Теперь создадим заголовок с помощью JavaScript.
      const heading = document.createElement("h1");
      heading.innerHTML = "Hello World from JS!"

      // Теперь нам нужно сделать так чтобы этот заголовок отображался в id="root"
      const root = document.getElementById("root");

      // Теперь добавляем дочерний элемент в наш div id="root".
      // Таким образом h1 станет дочерним элементом этого div id="root"
      root.appendChild(heading);
    </script>
  </body>
</html>
```

Теперь давайте посмотрим как создать базовый заголовок "Hello World" с помощью React.  
Сперва узнаем каким образом браузер понимает код `document.getElementById("root")`, `appendChild(heading)` и тд?  
Этот код браузер понимает с помощью суперспособностей, которые есть у браузеров.  
В браузерах есть движок JS, который выполняет JS код.

Но понимает ли браузер что такое React?  
Нет! Браузер не понимает код React.  
Поэтому нам нужно сначала добавить React в наш проект.

1. Мы можем добавить React в наш проект с помощью CDN (Сеть доставки контента),  
   по сути CDN — это место, где размещена библиотека React.  
   Мы просто загружаем React, добавляем его в наш код с помощью CDN.

```js
<script crossorigin src="https://unpkg.com/react@18/umd/react.development.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
```

После того как мы добавили эти ссылки CDN в наш код, произошло нечто невероятное.  
Теперь в нашем проекте есть React.  
Какой бы код React я не написал в моем проекте, наш браузер будет его понимать.

Что происходит когда мы добавляем React в наш проект?  
У нас происходит кое-что интересное за кулисами.  
Если мы откроем console в браузере и напишем `React`, у нас появится такой код в console.

```js
{Children: {...}, Fragment: Symbol(react.fragment), Profiler: Symbol(react.profiler), Component: f }
```

Это просто кусок кода, который написан на JS разработчиками Facebook, и мы можем использовать React в нашем проекте.

Почему у нас две ссылки CDN для React в нашем проекте?

1. **`react.development.js`** — это ядро React, это основной файл React.
2. **`react-dom.development.js`** — это `react-dom`, который используется для операций с DOM.  
   Он нам нужен для изменения модели документа DOM (Document Object Model).

Почему React создал разные файлы?  
Разве мы не могли бы обойтись одним файлом и объединить всё в нём?  
Нет! Почему?

React работает не только в браузерах — React работает также на мобильных устройствах (React Native).  
Есть ещё разные виды React, есть разные типы мест, где используется React.  
Поэтому есть разные функции и методы, которые использует React в браузерах или в других местах.

## Создание заголовка с помощью React

Теперь давайте приступим к написанию заголовка в React.

```js
// Теперь создадим элемент заголовка с помощью React.
const heading = React.createElement("h1", {}, "Hello World from React!");

// Теперь нам нужно сделать так, чтобы этот заголовок отображался в id="root" в качестве дочернего элемента.
// Сперва нам нужно создать root, чтобы React мог делать манипуляции с DOM (рендерить DOM).
// Когда мы хотим создать root для React, это уже работа второй ссылки CDN для работы с react-dom.
const root = ReactDOM.createRoot(document.getElementById("root"));

// Таким образом h1 станет дочерним элементом этого div id="root"
root.render(heading);

/*
Мы можем написать весь код React в нашем HTML-разметке внутри тега <script>,
либо мы можем написать этот код React в отдельном файле 
и подключить его для HTML-разметки <script src="react.js">
*/
```

### Для чего предназначен этот пустой объект?

```js
React.createElement("h1", {}, "Hello World from React!");
```

Этот объект — место, где вы задаёте атрибуты своим тегам.  
Например, моему тегу `h1` нужны идентификаторы (`id`), я могу сделать вот так:

```js
React.createElement(
  "h1",
  { id: "heading", xyz: "abc" },
  "Hello World from React!"
);
```

Допустим, мне нужно добавить немного CSS-стилей.  
Давайте изменим цвет нашего заголовка `h1`.  
Сперва подключаем CSS-стили в нашем HTML-разметке:  
`<link rel="stylesheet" href="styles.css">`.  
Пишем стили CSS в этом файле `styles.css` для нашего заголовка `h1#heading`.

```css
#heading {
  color: red;
}
```

Мы создали элемент с помощью React, вот так:

```js
const heading = React.createElement(
  "h1",
  { id: "heading", xyz: "abc" },
  "Hello World from React!"
);
```

Что это на самом деле?  
Если мы выведем в console этот элемент, что будет выведено?

```js
console.log(heading);

// Мы увидим объект
{
  $$typeof: Symbol(react.element),
  type: 'h1',
  key: null,
  ref: null,
  props: {id: 'heading', xyz: 'abc', children: 'Hello World from React!'},
  _owner: null,
  [[Prototype]]: Object
};
```

Мы видим, что в console выводится объект, а не тег `h1`.  
Что такое React-элемент?  
Это обычный объект JavaScript.

Когда мы вызываем `root.render(heading)`,  
мы передаём в этот React-объект идентификатор `div#root` — JavaScript-объект.  
Эта функция `render(heading)` берёт этот JavaScript-объект,  
создаёт тег `h1`, который понимает браузер, и помещает его внутрь тега `div#root`.  
Этот метод `render()` преобразует JavaScript-объект в тег `h1` и помещает его в DOM.

## Создание вложенной структуры в React

Мы только что создали один элемент в React для HTML-разметки.  
Но HTML-структура — это не просто один или два тега, верно?  
Чаще всего наша HTML-разметка не одноуровневая, а вложенная,  
например, вот так:

```html
<div id="parent">
  <div id="child">
    <h1>Я тег H1</h1>
    <h2>Я тег H2</h2>
  </div>
  <div id="child2">
    <h1>Я тег H1</h1>
    <h2>Я тег H2</h2>
  </div>
</div>
```

Каким образом мы создаём такую HTML-разметку в React?  
Давайте посмотрим, как создать вложенные элементы в React.

```js
// Сперва мы создадим родительский элемент `parent`
const parent = React.createElement("div");

// Теперь присвоим ему идентификатор
const parent = React.createElement("div", { id: "parent" });

// Теперь создадим дочерний элемент child
const parent = React.createElement(
  "div",
  { id: "parent" },
  React.createElement("div", { id: "child" })
);

// Теперь создадим дочерний элемент h1 для дочернего элемента child
const parent = React.createElement(
  "div",
  { id: "parent" },
  React.createElement(
    "div",
    { id: "child" },
    React.createElement("h1", {}, "Я тег H1")
  )
);

// Теперь создадим брата или сестру для дочернего элемента h1.
// Для этого мы создадим массив для дочерних элементов.
const parent = React.createElement(
  "div",
  { id: "parent" },
  React.createElement("div", { id: "child" }, [
    React.createElement("h1", {}, "Я тег H1"),
    React.createElement("h2", {}, "Я тег H2"),
  ])
);

/*
Когда мы создаём два элемента на одном уровне h1 и h2 как дочерние элементы,
React выдаёт нам ошибку, что нужно указать key.
Эту ошибку мы будем решать в будущем.
*/

// Теперь мы создадим брату или сестре для дочернего элемента child
const parent = React.createElement("div", { id: "parent" }, [
  React.createElement("div", { id: "child" }, [
    React.createElement("h1", {}, "Я тег H1"),
    React.createElement("h2", {}, "Я тег H2"),
  ]),
  React.createElement("div", { id: "child2" }, [
    React.createElement("h1", {}, "Я тег H1"),
    React.createElement("h2", {}, "Я тег H2"),
  ]),
]);

root.render(parent);
```

Как мы видим, когда мы создаём вложенные элементы в React для HTML-разметки,  
выглядит это ужасно и трудно читаемо.  
Поэтому в React существует **JSX**.

## React — библиотека, а не фреймворк

Почему React называется библиотекой, а не фреймворком?  
Потому что React работает только в том месте, где вы указываете идентификатор какого-то тега,  
где должен рендериться React-код.  
То есть вы можете сделать так, чтобы React работал в боковой панели, в шапке или в подвале вашего веб-приложения.  
Просто находите идентификатор вашего тега и создаёте для него корневой React-элемент.  
Вот так:

```js
// В таком случае React работает только внутри данного тега `div#root`.
const root = ReactDOM.createRoot(document.getElementById("root"));

// React может работать независимо в небольшой части вашего приложения.
```

Но если мы напрямую в HTML-разметке внутри данного тега `div#root` добавим любой элемент,  
то он будет заменён React-кодом, как только программа прочитает CDN-ссылки, которые мы добавили  
в нашем HTML через тег `<script>`.

А что такое **фреймворк**?  
Фреймворк содержит множество разных вещей.  
Фреймворк требует, чтобы вы создали приложение целиком с помощью этого фреймворка.  
А React можно использовать уже в существующих приложениях.
